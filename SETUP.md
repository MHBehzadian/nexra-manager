# راهنمای راه‌اندازی nexra manager (قدم‌به‌قدم)

این راهنما از صفر تا شروع کمپین را پوشش می‌دهد.

---

## گام ۰ — پیش‌نیازها (یک‌بار)

۱. **API_ID و API_HASH**: از https://my.telegram.org → *API development tools*
۲. **BOT_TOKEN**: از [@BotFather](https://t.me/BotFather) → `/newbot`
۳. **ADMIN_ID**: آیدی عددی خودت از [@userinfobot](https://t.me/userinfobot)
۴. **کانال شماره‌ها** (بازا): بات را در این کانال **ادمین** کن و دسترسی
   *Edit Messages* را بده. (اکانت‌های تو باید عضو کانال باشند.)

> چرا بات باید ادمین باشد؟ چون علامت‌گذاری «Task» روی پیام شماره‌ها با ویرایش
> پیام کانال انجام می‌شود و فقط ادمینِ کانال می‌تواند پست‌ها را ویرایش کند.

---

## گام ۱ — نصب روی سرور لینوکس

### دستور تک‌خطی (Ubuntu/Debian)

```bash
sudo apt update && sudo apt install -y git && \
git clone https://github.com/MHBehzadian/nexra-manager.git /opt/telegram-automation && \
cd /opt/telegram-automation && bash scripts/install.sh
```

> ریپو خصوصی است؛ روی سرور یا با `gh repo clone MHBehzadian/nexra-manager
> /opt/telegram-automation` کلون کن، یا از SSH/توکن گیت‌هاب استفاده کن.

اسکریپت، پکیج‌های سیستمی، `.venv` و وابستگی‌ها را نصب می‌کند.

---

## گام ۲ — پیکربندی اولیه (نوشتن `.env`)

```bash
cd /opt/telegram-automation
. .venv/bin/activate
python main.py
```

ویزارد، `API_ID`، `API_HASH`، `BOT_TOKEN`، `ADMIN_ID` (و اختیاری `CHANNEL_ID`)
را می‌پرسد و در `.env` ذخیره می‌کند. وقتی پیام «Bot online» را دیدی، با `Ctrl-C`
خارج شو (سرویس systemd آن را دائمی اجرا می‌کند).

> جایگزین: `cp .env.example .env` و ویرایش دستی.

---

## گام ۳ — سرویس دائمی (systemd)

```bash
sudo cp deploy/nexra-manager.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now nexra-manager
journalctl -u nexra-manager -f      # مشاهده‌ی لاگ‌ها
```

اگر مسیر نصب غیر از `/opt/telegram-automation` است، `User`، `WorkingDirectory`
و `ExecStart` را در فایل سرویس ویرایش کن.

---

## گام ۴ — کار با بات (در تلگرام)

بات را باز کن و `/start` بزن. منوی اصلی:
**📊 وضعیت · 👤 اکانت‌ها · 📇 شماره‌ها · 🚀 کمپین · ⚙️ تنظیمات · ❓ راهنما**

### ۴٫۱ افزودن اکانت‌ها
👤 **اکانت‌ها → ➕ افزودن اکانت** → یک *نام سشن* بده → *شماره* را بفرست →
کد ورود را **با فاصله/خط تیره** بفرست (مثل `1-2-3-4-5`) → در صورت داشتن رمز
دومرحله‌ای، رمز را بفرست. اکانت خودکار در کانال هم عضو می‌شود.
(کد و رمز به‌صورت best-effort از چت حذف می‌شوند.)

### ۴٫۲ تنظیم کانال شماره‌ها
📇 **شماره‌ها → 🔧 تنظیم کانال** → `@username` یا `-100…` یا لینک را بفرست.
(یا از ترمینال: `python main.py set-channel @your_channel`)

### ۴٫۳ عضویت همه‌ی اکانت‌ها
📇 **شماره‌ها → 🔗 عضویت همه‌ی اکانت‌ها** (اگر از قبل عضو باشند مشکلی نیست).

### ۴٫۴ تنظیم مدیا (فوروارد به بات)
🚀 **کمپین → 🎙 تنظیم مدیا (فوروارد)** — سپس ویس(ها) و تصویر(ها) را همان‌جا در
چت خصوصی بات **بفرست یا فوروارد کن**؛ هر کدام در `data/media/` ذخیره می‌شود.
در پایان **✅ پایان و ذخیره** را بزن. (مجموعه‌ی قبلی هر بار پاک و جدید ساخته می‌شود.)

### ۴٫۵ خواندن شماره‌ها
📇 **شماره‌ها → 📥 خواندن شماره‌ها** — هر اکانت پیام‌های کانال را می‌خواند،
شماره‌های `+98…` را استخراج و در دیتابیس ذخیره می‌کند (پیام‌های دارای «Task» رد
می‌شوند).

### ۴٫۶ تنظیم زمان ارسال ویس (اختیاری)
🚀 **کمپین → ⏱ زمان ویس** → دو عدد دقیقه بفرست، مثل `15 120`
(پیش‌فرض ۱۵ دقیقه تا ۲ ساعت پس از سلام).

### ۴٫۶ب متن همراه ویس (اختیاری)
🚀 **کمپین → ✍️ متن همراه ویس** → یک متن بفرست تا کنار ویس برای هر نفر ارسال شود
(ترتیب: ویس → این متن → تصاویر). «-» برای حذف.

### ۴٫۷ کانال گزارش (اختیاری)
🚀 **کمپین → 📮 تنظیم کانال گزارش** → آیدی یک کانال دیگر را بده تا گزارش‌ها و
بکاپ‌ها آنجا هم ارسال شوند. (بات باید در آن کانال ادمین باشد. «-» برای حذف.)
یا از ترمینال: `python main.py set-report-channel @your_report_channel`

### ۴٫۸ شروع کمپین (با تأیید تو)
🚀 **کمپین → ▶️ شروع** → صفحه‌ی تأیید را می‌بینی → **✅ بله، شروع کن**.

جریان کار:
- شماره‌ها نوبتی بین اکانت‌ها پخش می‌شوند؛ **۱ دقیقه فاصله** و هر شماره با
  **اکانت متفاوت**.
- به هر نفر اول یک **سلام رندوم**، بعد از **۱۵ دقیقه تا ۲ ساعت** ویس‌ها و تصاویر.
- روی پیام کانال: `Task ✅` (سلام رفت) → `Task ✅✅` (ویس رفت) → یا `Task ❌ علت`.
- فقط **۶ صبح تا ۱۲ شب به وقت ایران** ارسال می‌شود.
- اگر اکانتی موقتاً **محدود** شد (مثل PeerFlood): علت روی پیام نوشته می‌شود،
  شماره به اکانت دیگر می‌رود، و اکانت ~۴ ساعت **استراحت** می‌کند و بعد خودکار
  به چرخه برمی‌گردد (حذف نمی‌شود). فقط اکانت **بن/غیرفعال** برای همیشه خارج می‌شود.

---

## گام ۵ — گزارش و بکاپ

- هر شب ساعت ۲۳ (به وقت ایران) گزارش روزانه + بکاپ (`app.db` و `accounts.json`)
  برای ادمین ارسال می‌شود.
- دستی: 🚀 **کمپین → 💾 بکاپ الان** / **📈 گزارش الان**.
- وقتی صف شماره‌ها خالی شود، به ادمین اطلاع داده می‌شود.

---

## دستورهای مفید سرور

```bash
sudo systemctl restart nexra-manager   # ری‌استارت
sudo systemctl stop nexra-manager       # توقف
journalctl -u nexra-manager -f          # لاگ زنده
git -C /opt/telegram-automation pull && sudo systemctl restart nexra-manager  # آپدیت
```

> بعد از آپدیت، اگر ستون‌های دیتابیس تغییر کرده باشند، مهاجرت‌های سبک به‌صورت
> خودکار هنگام بالا آمدن اعمال می‌شوند.
